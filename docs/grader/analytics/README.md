# Repository Grader and Analytics

## Purpose

This package is a repeatable, evidence-backed review of the finance tracker. It preserves financial truth, server-blind finance storage, migration safety, and operational density while identifying remediation work.

## Current Baseline

The revision-specific baseline is [58/100](baseline-scorecard.md), with confirmed static evidence and explicitly bounded runtime uncertainty.

## Canonical Sources

[evidence-ledger.md](evidence-ledger.md) is the sole factual finding registry. [methodology.md](methodology.md) owns scoring and review rules. The scorecard, perspective reviews, domain reviews, and roadmap are derived views; they must not alter a finding's facts, severity, confidence, or status.

## Remediation Roadmap

Use the [master plan](roadmap/master-plan.md), [dependency map](roadmap/dependency-map.md), and [acceptance criteria](roadmap/acceptance-criteria.md). The reusable [rubric](scorecards/rubric.md) and [review template](scorecards/review-template.md) support later reviews.

## Regrading Workflow

Capture the revision, refresh static and runtime evidence, retain existing IDs, register new findings, attach resolution evidence, recalculate scores and caps, refresh derived reports and roadmap coverage, and publish the score delta.
